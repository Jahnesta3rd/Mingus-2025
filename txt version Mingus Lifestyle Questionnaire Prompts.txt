Create a PostgreSQL database schema for an AI Job Impact Calculator with the following requirements:

1. **ai_job_assessments table:**
   - id (UUID, primary key)
   - user_id (UUID, foreign key to users table, nullable for anonymous assessments)
   - job_title (VARCHAR(255))
   - industry (VARCHAR(100))
   - experience_level (VARCHAR(20))
   - tasks_array (JSONB) // Selected daily tasks
   - remote_work_frequency (VARCHAR(20))
   - ai_usage_frequency (VARCHAR(20))
   - team_size (VARCHAR(20))
   - tech_skills_level (VARCHAR(20))
   - concerns_array (JSONB) // AI-related concerns
   - first_name (VARCHAR(100))
   - email (VARCHAR(255))
   - location (VARCHAR(100))
   - automation_score (INTEGER)
   - augmentation_score (INTEGER)
   - overall_risk_level (VARCHAR(20)) // 'low', 'medium', 'high'
   - assessment_type (VARCHAR(50)) DEFAULT 'ai_job_risk'
   - completed_at (TIMESTAMP)
   - created_at (TIMESTAMP DEFAULT NOW())

2. **ai_job_risk_data table:** (Static reference data)
   - id (UUID, primary key)
   - job_keyword (VARCHAR(100))
   - automation_base_score (INTEGER)
   - augmentation_base_score (INTEGER)
   - risk_category (VARCHAR(20))
   - industry_modifiers (JSONB)
   - created_at (TIMESTAMP DEFAULT NOW())

3. **ai_calculator_conversions table:**
   - id (UUID, primary key)
   - assessment_id (UUID, foreign key)
   - conversion_type (VARCHAR(50)) // 'email_signup', 'paid_upgrade'
   - conversion_value (DECIMAL(10,2))
   - converted_at (TIMESTAMP)

Include appropriate indexes for email, job_title, industry, and completed_at fields.
